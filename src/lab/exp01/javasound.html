Enabling sound recording and playback using java applets

The experiments in SSP-Speech Signal Processing Virtual Lab are java applets that run in a browser. You need a java plugin to view these applets. Playback of sound works by default. You need to set some java security policies on your system to enable recording. Following are the instructions for doing the same:

For LINUX (Ubuntu and firefox)

As of now the support is only for Linux (Ubuntu) and firefox browser. The instructions would be very similar for other flavours of Linux or Windows and other open source browsers such as Opera or Google-Chrome. The expts do not work on IE-9.
  1. Install Sun Java-6 on your linux m/c. This is essential to enable java plugins on your browser. This step requires sys-admin privilege.
    The essential packages are:
    1. sun-java6-jre
    2. sun-java6-plugin

    If you use synaptic package manager, add the ubuntu partner repository, reload the package list, and select the above two packages. The dependencies are automatically selected. Install/Apply.

    Other java packages/plugins such as openjdk-6-jre (default jre) and icedtea6-plugin may or may not work. Try them out if they are already installed. Uninstall icedtea6-plugin to ensure there is no clash between this and sun-java6-plugin.

    Close all browser windows, restart the browser and check that the java applets (expts) are visible and you can playback audio.

  2. To record audio from your sound cards using the java applets, you need to set some permissions (java security settings) on your system. You just need to add a line to a file named "java.policy" normally found at these paths:

    /etc/java-6-sun/security/java.policy
    or
    /usr/lib/jvm/java-6-sun/jre/lib/security/java.policy

    Add the following line to the bottom of this file (last but one line) just before the last line which typically has a '};':

    permission javax.sound.sampled.AudioPermission "record";

    A sample java.policy file can be downloaded here for reference.

    If you do not have sys-admin privileges on your system, you can just download the java.policy and store it in your home directory or in the folder .java/deployment/security/ in your home directory.

    On windows systems the procedure is very similar except for the location of this java.policy file. You should find a sun-java-policy editor from your start-menu. You just need to add the above line.

  3. Close all browser windows, restart the browser and check if recording of sound works on your system.

Ofcourse you need to have a microphone and connect it to the proper jack (normally pink/red colored for mic or recording, green for playback or speakers), check the audio or sound card settings for recording to work.
